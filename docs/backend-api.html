API Reference

brAIn.md MCP Server Backend API

This document lists every backend API endpoint currently exposed by the Next.js server, how the UI should call it, which parameters it accepts, and what each route returns. It is intended as the human-friendly companion to /api/openapi.

Base paths
/api and /api/v1
Default auth cookie
brain_session unless overridden by env
Machine-readable spec
GET /api/openapi
Versioning
/api/v1/* rewrites to /api/*

How To Read This Doc

  • GET routes use query parameters when they need filters or scopes.
  • POST, PUT, and DELETE routes expect JSON request bodies unless the section says otherwise.
  • Unless noted, successful responses are JSON and errors follow { "error": "..." }.
  • Session-protected routes use the brain_session cookie set by login/register.
  • Token-management routes operate on the current tenant unless the route includes a specific {tokenId}.

Versioning And Response Rules

The app exposes a compatibility surface at /api and a versioned surface at /api/v1. The proxy.ts rewrite maps /api/v1/* to the same handlers, so the UI can pin to a versioned root without changing backend behavior.

Recommended UI convention: prefer /api/v1 for new code paths, but keep /api support for older links or local tooling.
All mutation endpoints return JSON error objects on failure. The UI should always check response.ok before assuming the returned JSON is a success payload.

GET /api

FieldValue
AuthNone
PurposeReturn the top-level API index and versioned root.
Request parametersNone
Response{ name, api_version, versioned_root, routes[] }

The routes array currently lists the core UI-facing endpoints under the version root.

{
  "name": "brAIn.md MCP Server API",
  "api_version": "v1",
  "versioned_root": "/api/v1",
  "routes": ["/health", "/vault/locks", "/vault/usage", "/vault/reindex", "/tokens", "/users", "/openapi"]
}

POST /api/search

FieldValue
AuthSession cookie required
PurposeRun semantic search against the tenant vault.
Content-Typeapplication/json
Required body fieldsquery
Optional body fieldstop_k or topK
Response{ tenant_id, query, top_k, matches[] }

The server caps top_k at 50 and ignores matches that fall outside the current token's readable roots.

Common Error Shape

Unless a route is returning a special case like the OpenAPI document, errors use a small JSON payload.

{
  "error": "Human readable message"
}
Status Typical Meaning
400Missing or invalid request data.
401Session missing or invalid.
403Authenticated but not permitted.
404Unknown file id or token id.
500Unexpected backend failure.

Source of truth for machine-readable clients: /api/openapi. Human-friendly UI docs live here.
